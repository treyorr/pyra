# Open-Source Readiness Checklist

Use this checklist before public launch or announcement.

## Product Readiness

- [ ] Active roadmap milestone acceptance criteria are satisfied.
- [ ] Required integration and contract tests pass in CI.
- [ ] Known P0/P1 defects are resolved or explicitly accepted.
- [ ] Error and JSON command contracts are stable and documented.

## Repository Readiness

- [ ] `README` explains install, quickstart, and core commands.
- [ ] License file is present and intentional.
- [ ] Contribution guidelines are present (`CONTRIBUTING.md`).
- [ ] Security policy/contact is present (`SECURITY.md`).
- [ ] Issue/PR templates are present.
- [ ] Roadmap and project direction docs are current.

## Engineering Hygiene

- [ ] Default test command and expected local workflow are documented.
- [ ] CI workflows are green on default branch.
- [ ] No secret material in tracked files.
- [ ] Release/versioning policy is documented.

## Announcement Readiness

- [ ] Public positioning matches current product reality.
- [ ] Known limitations are documented honestly.
- [ ] Launch post includes supported workflows and guardrails.
- [ ] Support and triage plan exists for first user wave.
