# Backlog Workspace

Store initiative-specific planning artifacts here.

Recommended structure:

`docs/backlog/<initiative-id>/`

Suggested files:

- `idea.md` (from `docs/templates/idea-brief-template.md`)
- `execution-roadmap.md` (generated from idea brief)
- `status.md` (from `docs/templates/status-template.md`)
- `release-readiness.md` (results from release audit prompt)

Keep initiative docs focused and evidence-driven. Avoid mixing unrelated work in
the same initiative directory.
